# Zen Mode — Design Spec

**Date:** 2026-06-28
**Status:** Approved for planning

## Summary

Add a relaxed, endless "Zen" game mode to Fruit Fury. Unlike Classic mode (which has lives, bomb penalties, and level progression), Zen mode removes failure states so players can enjoy pure slicing without pressure. The goal is a calming, satisfying experience — "fruit ninja meditation."

## Goals

- Provide a no-stress mode for casual play and practice.
- Remove lives/miss penalties so the game never ends until the player quits.
- Keep the core slicing loop identical — same physics, same blade, same fruit art.
- Include a soft timer (default 90s) for players who want a gentle goal, with an optional endless toggle.

## Non-Goals

- Arcade mode with power-ups (deferred).
- Multiplayer or networking.
- Social features or leaderboards.

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Lives | **None** — no life system in Zen | Removes all failure states |
| Bombs | **Hidden** — `bombChance: 0` in Zen level config | Config-driven, no spawner code change |
| Hearts | **Hidden** — `heartChance: 0` and `goldenHeartChance: 0` in Zen config | No healing needed when no lives |
| Missed fruit | **Not tracked** — `missFruit()` no-ops in Zen mode | No penalty for missing |
| Scoring | **Same system** as Classic | Players still get combo chain multiplier and points |
| Timer | **90s default**, toggle to endless | Gives high score meaning; endless for practice |
| Level progression | **Disabled** — fixed at Level 3 | Relaxed difficulty, no ramping pressure |
| Combo window | **Wider** (400ms vs 220ms) | Relaxed play style deserves more combo credit |
| Slow-mo fruit | **Enabled** (increased chance via config) | Adds variety and fun moments |
| High score | **Separate key** (`fruitFury.highScore.zen`) | Zen scores shouldn't dominate Classic "Best" |

## Architecture

### Where mode lives
- **`GameMode` type** added to `GameState` in `src/engine/scoring.ts` (not a separate module).
- `GameState` gets a `mode: 'classic' | 'zen'` field, initialized to `'classic'`.
- Existing methods become mode-aware via early-return guards:
  - `missFruit()`: if `mode === 'zen'`, return immediately (no life loss).
  - `sliceBomb()`: if `mode === 'zen'`, return immediately (no penalty).
  - `checkLevelUp()`: if `mode === 'zen'`, return false (no progression).
  - `heal()`: if `mode === 'zen'`, return 0 (no-op).

### Config changes
- Add `GameMode` to `config.ts`:
  ```ts
  zen: {
    level: 3,           // fixed level for zen
    bombChance: 0,      // no bombs
    heartChance: 0,     // no heal fruits
    goldenHeartChance: 0, // no golden hearts
    slowMoChance: 0.08, // doubled from 0.04
    spawnIntervalMs: 1200, // medium difficulty
    comboWindowMs: 400,  // wider combo window
  }
  ```
- Add a Zen `LevelConfig` entry in `src/engine/levels.ts` or handle via config directly.

### Spawner changes
- **No code changes to `makeSpawn`** — it's a pure cumulative threshold roll.
- Zen mode passes a config object with `bombChance: 0`, `heartChance: 0`, etc.
- The cumulative roll naturally shifts bomb probability mass to fruit, which is correct behavior.
- Slow-mo chance is per-mode via config, not global.

### Timer logic
- New `TimerState` in `GameState` for Zen mode:
  - `timerActive: boolean`
  - `timeRemaining: number` (in ms, starts at 90000)
  - `timerRunning: boolean` (paused)
- Timer counts down during gameplay, pauses on ESC.
- When timer hits 0: show "Time's Up!" overlay, then transition to score screen.
- Endless toggle: set `timeRemaining` to a very large number (or skip timer entirely).

### HUD changes
- Renderer gets a `mode: 'classic' | 'zen'` prop.
- **Classic HUD** (unchanged): hearts, level progress bar, score, best, level.
- **Zen HUD**: 
  - Show: Score, Best (zen-specific key), Timer (if not endless)
  - Hide: Hearts, Level progress bar
  - Show: "ZEN" badge in top-right corner
- Level number: not shown in Zen (fixed, never changes).

### Menu changes
- Title screen gets "Zen Mode" button between "Classic" and "Settings".
- Button id: `'zen'`.
- Activation: sets `gameState.mode = 'zen'` before `beginCountdown()`.
- Timer toggle: accessible from Zen mode settings or title screen.

### Game loop changes
- `update()` in `Game`:
  - `missFruit()` → no-op in Zen (guard in `GameState`).
  - `sliceBomb()` → no-op in Zen (guard in `GameState`).
  - `checkLevelUp()` → always false in Zen (guard in `GameState`).
  - Timer decrement: if `mode === 'zen'` and timer active, `timeRemaining -= dt * 1000`.
- `endGameIfOver()`:
  - In Zen, `isOver` never becomes true via lives/bombs.
  - Timer expiry triggers a "soft end" (not game over): show score summary, return to title.
  - Save high score to Zen-specific key.

### State reset
- `resetGame()` in `Game`:
  - Takes a `mode` parameter: `resetGame('classic' | 'zen')`.
  - Sets `gameState.mode = mode`.
  - For Zen: initialize `gameState.level = 3`, `gameState.lives = 0` (irrelevant), timer state.
  - For Classic: unchanged behavior.

## File Changes

### New Files
- None (all logic goes into existing modules with mode guards).

### Modified Files
- `src/types.ts` — Add `GameMode` type: `'classic' | 'zen'`.
- `src/config.ts` — Add `CONFIG.zen` section.
- `src/engine/scoring.ts` — Add `mode` to `GameState`, mode guards in `missFruit`, `sliceBomb`, `checkLevelUp`, `heal`. Add `TimerState` for Zen.
- `src/engine/levels.ts` — Add Zen level config (or handle via `CONFIG.zen`).
- `src/game/game.ts` — Add `mode` to `Game`, update `resetGame()`, add timer logic, add Zen-specific `endGameIfOver()` path.
- `src/game/renderer.ts` — Add `mode` prop, mode-aware HUD rendering.
- `src/game/menus.ts` (or wherever menu is) — Add Zen button.

## Testing Plan

### Unit Tests (Vitest)
- `missFruit()` no-ops in Zen mode.
- `sliceBomb()` no-ops in Zen mode.
- `checkLevelUp()` returns false in Zen mode.
- `heal()` returns 0 in Zen mode.
- Timer counts down correctly.
- Timer pauses on pause state.
- Timer expiry triggers soft end.
- High score persists to Zen-specific key.
- Level stays fixed at 3 in Zen.

### Manual Playtest
- Verify no bombs or hearts spawn.
- Verify timer counts down.
- Verify "Time's Up!" overlay appears.
- Verify score saves correctly.
- Verify returning to title shows "ZEN" mode.
- Verify endless toggle works.
- Verify combo chain multiplier works with wider window.

## Acceptance Criteria

- [ ] Zen mode button on title screen
- [ ] No lives, no bombs, no hearts, no game over in Zen mode
- [ ] Score and combo system work identically (with wider combo window)
- [ ] Timer counts down (90s default) and triggers soft end
- [ ] Endless toggle works (no timer)
- [ ] HUD reflects Zen mode (no hearts, shows ZEN badge, shows timer)
- [ ] High score saved to separate key
- [ ] All existing Classic mode functionality unchanged
- [ ] Tests pass for Zen mode logic
- [ ] Build succeeds with no type errors

## Out of Scope

- Arcade mode with power-ups.
- Multiplayer or networking.
- Social features or leaderboards.
- Mobile/touch-first design (desktop webcam is the target).
- Audio changes for Zen mode (same SFX as Classic).
- Visual theme changes (same Classic Dojo art style).
